# دورة حياة المحتوى السريري

## 1. لماذا هذا الملف مهم

المحتوى السريري في السرايا لم يعد مجرد نصوص أو إعدادات.  
لدينا عناصر تشغيلية تؤثر مباشرة على المريض مثل:

- `Order Sets`
- `Clinical Pathways`
- `CDSS rules`
- `Drug interaction content`
- `Critical values`
- `Consent templates`
- `Note templates`

لذلك يجب أن يدار هذا المحتوى كـ `controlled clinical content`.

## 2. أنواع المحتوى المشمولة

| النوع | المصدر الحالي داخل النظام | مستوى الخطورة |
| --- | --- | --- |
| Order Sets | order-sets | عالٍ |
| Clinical Pathways | clinical-pathways | عالٍ |
| CDSS rules | cdss + pharmacy | عالٍ |
| Drug interaction content | schema + cdss | عالٍ |
| Critical values | lab/vital critical values | عالٍ |
| Consent templates | ConsentForm | متوسط إلى عالٍ |
| Note templates | note-templates | متوسط |

## 3. الحالات الرسمية للمحتوى

كل عنصر محتوى يجب أن يمر بالحالات التالية:

1. `Draft`
2. `In Review`
3. `Approved`
4. `Published`
5. `Retired`

ولا يسمح بتجاوز هذه الدورة إلا في تغييرات الطوارئ.

## 4. البيانات الإلزامية لكل عنصر محتوى

كل عنصر محتوى يجب أن يحمل على الأقل:

- اسم إنجليزي وعربي إن وجد
- نوع المحتوى
- التخصص/الخدمة المالكة
- المالك السريري
- المراجع السريري
- مراجع السلامة
- رقم الإصدار
- تاريخ الاعتماد
- تاريخ بداية السريان
- تاريخ المراجعة القادمة
- سبب التغيير
- المراجع العلمية أو المرجعية المعتمدة
- مستوى الخطورة

## 5. السياسة الخاصة بالإصدار Versioning

### القاعدة الأساسية

- لا يجوز تعديل المحتوى المنشور عالي الخطورة بشكل صامت.
- أي تغيير مؤثر يجب أن ينتج إصدارًا جديدًا.

### التطبيق المطلوب في السرايا

بالنسبة إلى `Order Sets` و`Clinical Pathways`:

- يحفظ الإصدار المنشور كنسخة مرجعية ثابتة.
- ينشأ التعديل التالي كنسخة جديدة `draft`.
- لا ينتقل الإصدار الجديد إلى `published` إلا بعد الاعتماد.
- يحتفظ النظام بتاريخ المقارنة بين الإصدارين.

## 6. ضوابط النشر

قبل النشر يجب التأكد من:

- اكتمال مراجعة سريرية
- اكتمال مراجعة سلامة
- اكتمال مراجعة تكاملية وتقنية
- اجتياز الاختبارات المطلوبة
- وجود خطة rollback للمحتوى عالي الخطورة
- وجود خطة اتصال للمستخدمين إذا كان الأثر تشغيليًا

## 7. ضوابط الإيقاف أو السحب

يجب أن يملك كل عنصر عالي الخطورة آلية:

- `suspend`
- `retire`
- `emergency disable`

مثال:

- قاعدة دوائية تبين أنها ترفع false positives بشكل خطر
- `order set` يحتوي جرعة خاطئة
- مسار علاجي أصبح غير مطابق للبروتوكول

## 8. المتطلبات التقنية المستهدفة

هذه الدورة تتطلب لاحقًا تنفيذًا تقنيًا واضحًا داخل المنتج:

- جداول إصدارات للمحتوى السريري
- حالات workflow
- سجل اعتماد وتوقيع
- مقارنة بين الإصدارات
- سجل نشر وسحب
- ربط مع `AuditLog`

## 9. التحقق بعد النشر

لكل محتوى عالي الخطورة منشور:

- مراجعة أداء خلال 7 أيام
- مراجعة استخدام خلال 30 يومًا
- مراجعة جودة/أثر خلال 90 يومًا

## 10. أمثلة على محتوى يجب منعه من النشر المباشر

- تغيير جرعة أو route أو frequency في `order set`
- تعديل عتبات `critical values`
- تعطيل قاعدة `drug interaction`
- تعديل مسار علاجي يستخدم داخل IPD
- تعديل نموذج `consent` جراحي
